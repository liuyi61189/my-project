- **根因（两层叠加）**：

- **修复（双保险，兼容两个 key）**：
  - 前端 `TestCaseManager.vue` 的 `doDumpPage` 改为 `response.data.candidates || response.data.elements || []`。
  - 同时 `docker compose build backend && docker compose up -d backend` 重建后端镜像，使运行代码与磁盘 `views.py` 一致（返回 `elements`）。
- **教训**：前后端约定字段名时务必一致；后端改了响应结构却没同步前端时，前端至少要兼容历史 key。后端改 `apps/` 后不能只 `docker restart`，必须 `docker compose build` 重镜像（未挂卷）。

## 录制点击无效修复（用户反馈：点击后弹窗不关闭）
- **现象**：录制模式下截图点选元素（如日历弹窗中的日期）→ 点"记录点击"→ 步骤记录成功但设备上弹窗没关。
- **根因**：Appium 的 `el.click()` 对 Android 很多控件（RecyclerView 容器、自定义 View、WebView 元素等）会"返回成功但实际无效果"。而 `dump_page_elements` 已返回元素的屏幕坐标 (`center_x`, `center_y`)，但之前 `record_action` 构造 `element_data` 时丢弃了坐标信息，`_do_click` 只用了不可靠的 `el.click()`。
- **修复（3 处）**：
  1. `views.py` 的 `record_action`：`element_data` 新增 `center_x`/`center_y` 字段透传前端坐标。
  2. `appium_engine.py` 的 `_do_click`：优先使用 `TouchAction.tap(x=cx, y=cy)` 坐标点击，失败回退 `el.click()`。
  3. `_do_double_tap` / `_do_long_press` 同样改为坐标优先。
- **部署**：需重建后端镜像（appium_engine.py 改了）+ 前端构建。

## 录制用例"步骤无元素 + 跑不起来"根因与坐标回放方案（重要）
- **用户反馈**：录制生成用例后，步骤里看不到元素；生成的用例也跑不起来。
- **根因A（跑不起来）**：回放链路 `test_executor.py` 构建 `step_data['element']` 时只带了 `locator_value`/`locator_strategy`，**完全没带 center_x/center_y**；而 `Element` 模型本来也没存坐标。于是 `_do_click` 退化成不可靠的 `el.click()`，`locator`（resource-id / xpath）在目标 App 上常匹配失败 → `NoSuchElementException` → 用例失败。
- **根因B（步骤无元素 UI）**：前端 `selectTestCase` 之前已加 `loadElements(page_size=999)` + 兜底注入，但用户可能未重建前端；DB 里 Element 实际已正确关联。
- **方案（坐标回放，最稳）**：
  1. `models.py` 的 `TestCaseStep` 新增 `center_x`/`center_y`（IntegerField, null）；新增迁移 `0003_testcasestep_coords.py`（容器启动 `docker_start.py` 会自动 makemigrations+migrate 兜底）。
  2. `generate_recording_case`：把录制时 `ed.center_x/center_y` 写入 `TestCaseStep`；并自动在最前插入一步 `launch_app`（用 `sess['app_config'].package_name`），让回放从干净起点开始。
  3. `test_executor.py`：构建 `step_data['element']` 时带上 `step.center_x/center_y`。
  4. `_do_click` 已有"坐标优先 `driver.tap([(x,y)])`"逻辑 → 现在回放能真正命中屏幕坐标，不依赖 locator。
- **结论**：对动态 UI 的录制工具，坐标回放比 locator 可靠得多。后续如要支持不同分辨率设备，可再加分辨率归一化。
- **部署**：后端必须 rebuild（models/migrations/views/test_executor 都改了）；前端也要 rebuild（selectTestCase 兜底此前已改）。
