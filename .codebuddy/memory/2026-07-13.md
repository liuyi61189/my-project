## 拆解结果白屏崩溃修复（TypeError: Cannot read properties of undefined (reading 'length')）
- 根因1：模板 `section.rows.length` / `section.headers.length` 在 section 缺字段时崩溃
- 根因2（关键）：`parsedAnalyzeSections` 返回的 section 可能缺 `rows`/`headers`，且 `exportAnalyzeToMarkdown` 内 `s.rows.length`/`s.headers.length` 同样无守卫
- 修复：① 模板两处加 `section.rows &&` / `section.headers &&` 守卫；② `exportAnalyzeToMarkdown` 内 `s.rows &&`/`s.headers &&` 守卫；③ `parsedAnalyzeSections` computed 对每个 section 补 `rows:[]`/`headers:[]` 默认值；④ 多列表 `<table>` 加 `v-if="section.rows && section.rows.length"`；⑤ 纯文本段加 `v-if="section.text"` 守卫
- 部署方式踩坑：本次 `docker compose build --no-cache frontend` 仍命中旧缓存，最终用 `docker exec testhub-frontend rm -rf assets index.html` + `docker cp dist/.` 直接复制才生效（验证 MD5 一致才确认生效）
- 用户确认简化版（纯 markdown 渲染）能看到内容 → 确认崩溃在结构化渲染路径

## 知识库改回绑定挂载（实时同步最终落地，2026-07-13 晚）
- 用户要求把 docker-compose 知识库挂载从命名卷改回**绑定挂载**以实现本地文件实时同步
- 踩坑：Docker Desktop 把 Windows 路径翻译成 `/run/desktop/mnt/host/c/...` 后会对父目录 `/run/desktop/mnt/host/c` 执行 `mkdir` 报 `file exists`，容器卡 `Created` 起不来。试过 `C:/`、`/c/`、`/./` 三种写法都失败，**与路径格式无关**
- 最终修法：`wsl --shutdown` 重启 WSL2 里的 Docker 引擎清除该坏状态 → 引擎恢复后 `docker-compose up -d backend` 按新配置（`./knowledge-base:/app/knowledge-bases`）重建，挂载类型变为 `bind C:\TestHub\knowledge-base -> /app/knowledge-bases`，**实时同步验证通过**（本地建 `_sync_test.txt` 容器内立即可见）
- 注意：`wsl --shutdown` 会短暂中断所有容器（testhub 全套 + docker-* 栈），但带 `restart: always` 的会自动恢复（已确认全部 Up）
- 教训：Windows 上 Docker Desktop 绑定挂载 Windows 路径偶发此 bug，重启引擎是唯一可靠修法；重启后 compose 里的 bind 配置即可正常生效
