## 二、不影响的功能（保留原行为）

以下功能**行为完全不变**：

- 登录接口 `/api/auth/login/` — 仍为 `AllowAny`（必须允许匿名登录）
- 注册接口 `/api/auth/register/` — 仍为 `AllowAny`（必须允许匿名注册）
- 所有 `@csrf_exempt` 装饰器 — 保留（JWT 认证不受 CSRF 影响）
- Admin 后台 — CSRF 保护恢复（`@csrf_exempt` 不影响 admin 路径）
- 所有业务逻辑（用例管理、评审、执行、API 测试、UI 自动化）— 未改动

## 三、行为变化的功能

| 功能 | 变化 | 影响 |
|---|---|---|
| 未登录访问需求分析接口 | 之前：返回数据（冒充超管） → 现在：返回 401 | 前端必须确保用户已登录才调用这些接口 |
| 普通用户管理用户列表 | 之前：任何登录用户可增删用户 → 现在：仅管理员 | 前端用户管理页普通用户会收到 403 |
| Docker 部署 | 之前：明文密码 + Windows 路径 → 现在：需要 `.env` 文件 | 首次部署需 `cp .env.example .env` 并填入密码 |
| 前端构建 | 之前：plugin-vue 4 + Vite 7 不兼容 → 现在：plugin-vue 6 | `npm install` 后构建应正常 |
