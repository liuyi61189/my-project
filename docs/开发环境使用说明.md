# TestHub 隔离开发环境

## 安全边界

开发环境使用独立的 Compose 项目、端口、网络和数据卷，不连接生产数据库。

- 生产前端：8080；开发前端：18080。
- 生产后端：8000；开发后端：仅本机 18000。
- 开发 MySQL：仅本机 13306。
- 开发 Redis、媒体、日志、知识库和 Chroma 均使用独立数据卷。
- 后端源码只挂载到开发容器，生产容器不会自动加载宿主机修改。
- APP profile 默认不启动，防止空数据库误启动业务服务。

## 第一次准备

生成仅供开发使用的环境文件：

    powershell -ExecutionPolicy Bypass -File scripts/init_dev_env.ps1

检查配置：

    python scripts/check_configuration.py .env.dev --ignore-process-env

只启动开发 MySQL 和 Redis：

    docker compose -p testhub-dev --env-file .env.dev -f docker-compose.dev.yml up -d mysql redis

确认状态：

    docker compose -p testhub-dev --env-file .env.dev -f docker-compose.dev.yml ps

## 初始化数据库

本项目的历史迁移包含占位迁移，不能依靠 migrate 从空库完整重建表。开发库必须先导入一份结构基线。

推荐从生产数据库导出 schema only，不复制任何用户或业务数据。该操作应在低峰期执行，并将 SQL 文件放在受控临时目录。导入目标必须确认是 testhub-dev 的 MySQL 容器。

完成结构导入后再启动应用：

    docker compose -p testhub-dev --env-file .env.dev -f docker-compose.dev.yml --profile app up -d --build

开发地址：

    http://服务器IP:18080

## 日常命令

查看开发环境：

    docker compose -p testhub-dev --env-file .env.dev -f docker-compose.dev.yml ps

查看开发后端日志：

    docker compose -p testhub-dev --env-file .env.dev -f docker-compose.dev.yml logs -f backend

停止开发应用但保留数据：

    docker compose -p testhub-dev --env-file .env.dev -f docker-compose.dev.yml --profile app stop frontend backend

停止全部开发容器但保留数据：

    docker compose -p testhub-dev --env-file .env.dev -f docker-compose.dev.yml stop

## 禁止事项

- 不对生产容器执行 docker cp。
- 不让开发后端连接 testhub-mysql。
- 不使用 docker compose down -v，除非明确要删除开发数据库。
- 不使用 docker system prune。
- 不在开发环境配置真实邮件、Webhook 或生产 AI 密钥。
- 不把 .env.dev 提交到 Git。
