# 远程手机接入教程（多人共享真机）

本文档面向**使用 TestHub 平台的测试人员**：你只需把手机通过 USB 插在自己电脑上，平台服务器就能远程控制你的手机进行录制 / 执行用例。**不需要在服务器上插手机，也不需要在自己电脑装 Appium。**

---

## 一、整体架构

```
测试人员电脑 (如 192.168.8.71)          平台服务器 (192.168.8.75, Docker)
  │                                        │
  │  USB 线 ── 手机                         │  testhub-backend 容器
  │                                        │
  │  adb -a -P 5037 nodaemon server  ───→  │  adb -H 192.168.8.71 -P 5037
  │  (开放 5037 网络监听)                   │  通过该电脑的 ADB 控制手机
  │                                        │
  │  防火墙放行 5037 入站                   │
```

- 平台后端通过**你电脑的 ADB 服务（5037 端口）**控制手机
- **Airtest 录制 / 执行模式只需 ADB（5037）**，不需要 Appium（4723）
- 每台设备在平台登记时填写「电脑IP(ADB)」= 你电脑的 IP，平台据此找到正确的 ADB 服务

## 二、用户侧配置（Windows）

### 2.1 检查 ADB 是否已安装

```powershell
adb version
```

- 显示版本号（如 `Android Debug Bridge version 1.0.41`）→ 已安装，跳到 **2.3**
- 显示"不是内部或外部命令" → 没装，走 2.2

### 2.2 安装 ADB（三选一，推荐第 1 种）

**方式 1：用 Scoop（最省事，自动加 PATH）**

```powershell
# 没装 scoop 先装（需管理员 PowerShell）
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
irm get.scoop.sh | iex

# 安装 ADB
scoop install adb
```

**方式 2：用 Chocolatey**

```powershell
choco install adb -y
```

**方式 3：手动下载（不用包管理器）**

1. 浏览器下载 `https://dl.google.com/android/repository/platform-tools-latest-windows.zip`
2. 解压到 `C:\platform-tools\`
3. 把 `C:\platform-tools` 加进系统 PATH（设置 → 系统 → 关于 → 高级系统设置 → 环境变量 → Path → 新建）

装完**重开一个 PowerShell** 验证：`adb version`

### 2.3 手机开启 USB 调试

1. 手机：**设置 → 关于手机 → 连续点「版本号」7 次** 打开开发者模式
2. **设置 → 系统 → 开发者选项 → 开启「USB 调试」**
3. USB 线连上电脑，手机弹窗点 **「允许 / 始终允许」**
4. 验证连接：

```powershell
adb devices
```

看到类似 `20a1ae2d    device` 说明连上了（`20a1ae2d` 是你的手机序列号）

### 2.4 启动 ADB 网络服务（关键）

```powershell
# 以管理员身份打开 PowerShell，执行：
adb -a -P 5037 nodaemon server
```

> ⚠️ 这个命令会**一直占用当前窗口**（正常现象）。两种用法：
> - **临时测试**：就让它开着这个窗口别关
> - **长期后台**：另存为 `C:\start_adb.bat`，内容如下，双击运行即可（关窗口 = 停服务）
>   ```bat
>   @echo off
>   adb -a -P 5037 nodaemon server
>   ```

### 2.5 防火墙放行 5037（管理员 PowerShell）

```powershell
netsh advfirewall firewall add rule name="ADB_TCP_5037" dir=in action=allow protocol=TCP localport=5037
```

只放行平台服务器（如 `192.168.8.75`）更安全：

```powershell
netsh advfirewall firewall add rule name="ADB_TCP_5037_from_75" dir=in action=allow protocol=TCP localport=5037 remoteip=192.168.8.75
```

验证端口在监听：

```powershell
netstat -ano | findstr 5037
```

应看到 `0.0.0.0:5037` 或 `LISTENING` 状态。

## 三、平台侧登记设备

1. 浏览器打开平台 → **App自动化测试 → 设备管理**
2. 点左上角 **「🔄 刷新设备」** 按钮（后端会自动扫描你的 ADB 发现手机）
3. 列表里出现你的手机后，在 **「电脑IP(ADB)」** 列点「点击设置」
4. 填入**你这台电脑的 IP**（如 `192.168.8.71`），回车保存

> 查自己电脑 IP：`ipconfig | findstr IPv4`

---

## 四、验证（在服务器上测）

```powershell
# 在服务器 192.168.8.75 上执行
adb -H 192.168.8.71 -P 5037 devices
```

能看到你的手机序列号 = 全链路打通 ✅

---

## 五、开始录制 / 执行

1. 进 **App自动化测试 → 真机录制(旧版)**
2. 引擎下拉框选 **Airtest**（不是 Appium）
3. 选你的设备 + 应用 → 点「开始录制」
4. 录完点「继续录制」→ 「生成用例」→ 自动导出 `.py` 脚本

> Airtest 模式只走 ADB（5037），**不需要 Appium**。

## 六、快速排错

| 现象 | 解决 |
|------|------|
| `adb devices` 看不到手机 | 重插 USB、重开 USB 调试授权、换数据线 |
| 服务器 `adb -H <ip> devices` 连不上 | 检查防火墙规则、用户电脑 IP 是否变了、ADB 服务窗口是否还开着 |
| 平台上刷新不出设备 | 确认 `adb -a -P 5037 nodaemon server` 在跑 + 防火墙已放行 5037 + 设备「电脑IP(ADB)」列已填值 |
| 录制报 4723 错误 | 确认选的是 **Airtest 引擎**（不是 Appium），Airtest 只用 5037 |
| 「电脑IP(ADB)」列没值 | 在设备管理列表点该列「点击设置」填入你电脑 IP |
| 刷新显示"发现0个设备"但之前有 | 点刷新后后端会遍历所有已登记的 `adb_host` 去扫 ADB；确保该设备的**电脑IP(ADB)列有值**，且该IP的 ADB 服务正在运行 |

## 七、一键汇总（用户照抄版）

```powershell
# ① 管理员 PowerShell 执行（放行端口）
netsh advfirewall firewall add rule name="ADB_TCP_5037" dir=in action=allow protocol=TCP localport=5037

# ② 另开普通 PowerShell，启动服务（窗口保持打开）
adb -a -P 5037 nodaemon server

# ③ 验证本机
adb devices
```

然后去平台「刷新设备」→ 设置「电脑IP(ADB)」即可。

## 附：管理员 / 部署说明（仅供运维参考）

- 设备模型 `app_automation.AppDevice`（表 `app_automation_devices`）与 `ui_automation.AppDevice`（表 `app_devices`）均有 `adb_host` 字段（GenericIPAddressField）
- 录制 / 执行用例时，Airtest 引擎会读取设备的 `adb_host`，作为 `adb -H <adb_host> -P 5037` 的主机地址；为空则回退到 `host.docker.internal`
- `docker-compose.yml` 中 `host.docker.internal` 已改回 `host-gateway`（不再写死单台电脑 IP），多人的区分完全靠设备的 `adb_host` 字段
- 前端「设备管理」页面（`app-automation/devices/DeviceList.vue`）表格支持内联点击编辑「电脑IP(ADB)」列
